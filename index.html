Rails Best Practices Study by Miguel Regedor

by following best practices, a project's maintainability is increased.

It seems an obvious assumption, although it requires formal proof.

The objective of this project is to find a correlation between a project's quality and its usage of best practices.

Rails Best Practices Study is a tool (demo) serving of support for my Master's thesis.

About the tool

The tool is open source and the source code hosted on github. It consists of a Ruby on Rails application with two main parts:

The first part is a web interface where you can enter one or more links to github projects (Rails applications). The links are stored in the database for later processing. After concluded the assessment of a project, the report and best practices score is presented in this same web interface.

The second part is the a daemon that should be running in paralel (delayed job). The purpose of this process is to check for new links on the database, use them to clone the projects repositories, retrieve github details, run the analyses based on the ruby_best_practices gem upon them and store the report in the database.

Anyone is free to download this project and use without any limitations. A online instance of the tool (demo) is also available here. However at this moment, the daemon is only running once a week (this happens to avoid heroku hosting costs).

Dependencies

The project consists of a regular Rails application, the dependencies are listed in the Gemfile. The only uncommon thing is the need to use my altered version of Richard Huang's gem: rails_best_practices
